# FAQ

**Бот не отвечает после деплоя.**
– Проверить `BOT_TOKEN` в Variables (без `BOT_TOKEN=` в значении, только сам токен). Перезапуск деплой.

**/health показывает «пинг БД: FAIL».**
– Значит нет доступа к файлу БД. Проверить `DB_FILE` и права/volume. Для локали: `bot/data.sqlite3`.

**Как добавить администратора?**
– В `.env` указать `ADMIN_IDS=111,222` и перезапустите бота. В коде это читается как множество ID.

**Можно ли перенести историю с локали на прод?**
– Да. Скопировать `data.sqlite3` на сервер и указать путь в `DB_FILE`, либо импортировать CSV/Sheets через отдельный скрипт.

**Polling или Webhook?**
– По умолчанию polling — проще. На PRO-тарифах/больших нагрузках выгоднее webhook.

**Бот будет работать 24/7?**  
Да, при деплое на Railway/Render — без вашего ПК.

**Можно оплату/картой?**  
Да, через ЮKassa/CloudPayments/Telegram Stars (апсейл).

**Можно выгружать в Excel/таблицу?**  
Да: CSV-файлы и/или Google Sheets.

**Если вырубится интернет?**  
Заказы остаются в БД, бот поднимется и продолжит работу.
